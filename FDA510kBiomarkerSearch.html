Biomarker Search
openFDA Device Clearance Lookup
Uses the public openFDA device 510(k) API. No key: 1,000 requests/day. Add a free key for 120,000/day.
LDT search calls a small proxy Worker (NY's LDT site has no CORS support, so this tool can't reach it directly) — see worker/README.md in the repo for step-by-step deploy instructions (~5 min, no coding required).
Optional: "Check Measurand" on an FDA record reads the device's official Decision Summary PDF to confirm what it actually measures (same CORS problem as LDT — needs its own small proxy Worker, deployed the same way; see worker/README.md). Not every device has one.
Biomarkers
Terms are matched against device name and 510(k) summary text (openFDA has no dedicated "biomarker" field). Greek letters (α, β, …) are automatically spelled out (alpha, beta, …). If an exact phrase finds nothing, the search automatically widens — first dropping word order, then the Ig-class suffix — and flags the result as Broad match / Antigen-only match so you know it's less precise. "Anti-" is never dropped when the term has it — it distinguishes an antibody test from the antigen itself, so a real match must still contain it, though not necessarily the literal word "anti" ("antibody", "antibodies", "autoantibody", "autoantibodies", and "AB" all count too), and not necessarily in the same field as the antigen name — FDA splits one device's info across several fields, and it's common for the antigen abbreviation to appear in one and the antibody wording in another. For antigens known to appear in named multi-antigen panel kits (autoimmune liver, ANA/ENA, celiac, and similar profiles), 0 confirmed results also triggers a search for plausibly related panel devices, shown separately as N possible panel matches — these aren't confirmed (panels rarely name each individual antigen in FDA's structured data), so verify manually, e.g. via Check Measurand.
Terms are matched against the analyte/test name in the NY State Wadsworth Center CLEP "Approved Laboratory Developed Tests" database (any-word match, same as the site's own default). Requires an LDT proxy Worker URL configured in Settings (gear icon).
